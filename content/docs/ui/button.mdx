The **Button** component is one of the core interactive elements of PyColors UI.
It provides consistent styling, variants, accessibility, and flexible composition across marketing pages, dashboards, and applications.

This implementation extends the **shadcn/ui `button` primitive** and integrates smoothly with the PyColors design system.

---

## Import

```tsx
import { Button } from "@/components/ui/button";
```

### Available variants

| Variant     | Description                          |
| ---------- | ------------------------------------ |
| default    | Primary action button, high emphasis |
| secondary  | Neutral alternative to primary       |
| outline    | Bordered button for medium emphasis  |
| ghost      | Minimal UI footprint                 |
| destructive| Used for destructive actions         |
| link       | Text-style button for inline actions |

## API

### Props

| Prop      | Type          | Default   | Description                     |
| --------- | ------------- | --------- | ------------------------------- |
| variant   | ButtonVariant | "default" | Visual style of the button      |
| size      | ButtonSize    | "default" | Size of the button              |
| asChild   | boolean       | false     | Render using Radix Slot         |
| disabled  | boolean       | false     | Disable interaction             |
| className | string        | —         | Additional Tailwind classes     |

The Button extends the standard HTML button props:

```ts
React.ButtonHTMLAttributes<HTMLButtonElement>
```

### TypeScript types

The Button component exposes two typed props — `variant` and `size` — extending
the shadcn/ui `button` primitive. These types ensure design-system consistency
and catch incorrect values at compile-time.

```ts
/**
 * Visual styles available for the Button component.
 * Each variant corresponds to a semantic purpose
 * and maps to a specific set of Tailwind classes.
 */
export type ButtonVariant =
  | "default"        // Primary action button, high emphasis
  | "secondary"      // Neutral, less prominent
  | "outline"        // Bordered button for medium emphasis
  | "ghost"          // Minimal footprint, subtle action
  | "destructive"    // Destructive or irreversible actions
  | "link";          // Inline link-style button

/**
 * Available sizes for the Button component.
 * These values are aligned with the spacing scale of
 * the PyColors design system for consistent layout rhythm.
 */
export type ButtonSize =
  | "sm"      // Compact buttons, use in dense UI
  | "default" // Standard size – the recommended default
  | "lg"      // Larger visual weight for important actions
  | "icon";   // Square button for icon-only interactions

```

## Accessibility

- Use clear and descriptive labels for button text.
- Preserve keyboard focus styles (focus-visible) for accessible navigation.
- For icon-only buttons, provide an aria-label describing the action.
- Avoid using Button with only an icon and no accessible text or label.

---

## Design guidelines

- Use the default variant for primary actions on a page.
- Use secondary or outline for less prominent or secondary actions.
- Use destructive only for actions that are dangerous or irreversible.
