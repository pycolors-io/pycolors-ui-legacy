The **Input** component provides a consistent, accessible, and theme-aware text field for forms and UI surfaces.

It supports labels, helper text, validation errors, and optional icons on the left or right — making it suitable for dashboards, settings pages, authentication flows, and complex forms built with **React Hook Form + Zod**.

---

## Import

```tsx
import { Input } from "@/components/ui/input";
```

## Validation error

The `error` prop:

- Toggles the `hasError` visual state.
- Sets `aria-invalid` on the input.
- Updates `aria-describedby` to point to the error message.
- Displays the error message below the field.

### Right icon

Often used for password reveal/hide toggles, loading states, or inline validation.

<Tabs items={["Preview", "Usage", "Code"]}>
  <Tab value="Preview">
    <Preview>
      <Input
        type="password"
        label="Password"
        placeholder="••••••••"
        rightIcon={
          <Eye className="h-4 w-4" />
        }
      />
    </Preview>
  </Tab>

  <Tab value="Usage">
    ```tsx
    <Input
      type="password"
      label="Password"
      placeholder="••••••••"
      rightIcon={<Eye className="h-4 w-4 text-muted-foreground" />}
    />
    ```
  </Tab>


 <Tab value="Code">
    ```tsx
    "use client";

    import * as React from "react";
    import { Input } from "@/components/ui/input";
    import { Eye, EyeOff } from "lucide-react";

    export function PasswordInput() {
      const [visible, setVisible] = React.useState(false);

      return (
        <Input
          type={visible ? "text" : "password"}
          label="Password"
          placeholder="••••••••"
          rightIcon={
            <button
              type="button"
              onClick={() => setVisible(v => !v)}
              className="inline-flex items-center justify-center text-muted-foreground hover:text-foreground"
              aria-label={visible ? "Hide password" : "Show password"}
            >
              {visible ? (
                <EyeOff className="h-4 w-4" />
              ) : (
                <Eye className="h-4 w-4" />
              )}
            </button>
          }
        />
      );
    }

    export default PasswordInput;
    ```
  </Tab>
</Tabs>

Icons are wrapped with `pointer-events-none` inside the field container so they don’t interfere with focus or selection, while interactive controls (like the password toggle above) are rendered with proper semantics.

### Recommended usage

- Use `"md"` as the default size for most forms.
- Reserve `"sm"` for dense UIs such as tables, filters, or toolbars.
- Use `"lg"` for marketing pages, onboarding flows, or high-focus fields.

## API

### Props

| Prop        | Type                                    | Default | Description                                      |
| ----------- | --------------------------------------- | ------- | ------------------------------------------------ |
| `label`     | `string`                               | —       | Accessible label displayed above the input       |
| `helperText`| `string`                               | —       | Optional helper text shown below the field       |
| `error`     | `string`                               | —       | Validation error message                         |
| `leftIcon`  | `React.ReactNode`                      | —       | Icon rendered inside the field, on the left      |
| `rightIcon` | `React.ReactNode`                      | —       | Icon rendered inside the field, on the right     |
| `size`      | `"sm" \| "md" \| "lg"`                 | `"md"`  | Controls input height and padding                |
| `className` | `string`                               | —       | Additional Tailwind classes                      |
| `...props`  | `React.InputHTMLAttributes<HTMLInputElement>` | — | Native input props (type, placeholder, etc.)     |

### TypeScript types

```ts
export interface InputProps
  extends Omit<React.InputHTMLAttributes<HTMLInputElement>, "size"> {
  /**
   * Accessible label displayed above the input.
   * If omitted, ensure the field is still labeled
   * via an external <label> or aria-label.
   */
  label?: string;

  /**
   * Optional helper text rendered below the field.
   * Use this for hints or non-critical guidance.
   */
  helperText?: string;

  /**
   * Validation error message rendered below the field.
   * When provided, the input is marked as invalid and
   * the error message is linked via aria-describedby.
   */
  error?: string;

  /**
   * Icon rendered inline on the left side of the field.
   * Use for search icons, prefixes, or field context.
   */
  leftIcon?: React.ReactNode;

  /**
   * Icon or control rendered inline on the right side of the field.
   * Use for password toggles, clear buttons, or status indicators.
   */
  rightIcon?: React.ReactNode;

  /**
   * Visual density of the field.
   */
  size?: "sm" | "md" | "lg";
}
```

## Accessibility

- Uses `aria-invalid` when `error` is present.
- Automatically wires `aria-describedby` to helper/error text when available.
- Icons are visually inline but ignored by screen readers when static.
- Always pair `Input` with a visible label in forms for best accessibility.
- Preserve focus rings and keyboard navigation (Tab, Shift+Tab).

---

## Design guidelines

- Use clear, concise labels (1–3 words when possible).
- Prefer placeholder text as an example, not the only label.
- Keep helper text short; move complex explanations to adjacent content.
- Align inputs to an 8px grid using the provided spacing scale.
- Use icons with `className="size-4"` to stay consistent with the design system.
